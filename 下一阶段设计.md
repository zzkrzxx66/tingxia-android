## 1. 阶段目标

0.1 已完成本地听书的核心闭环：

```text
选择目录 → 扫描成书 → 章节连播 → 后台播放
→ 断点续听 → 倍速 → 快进快退 → 睡眠定时
```

0.2 的目标不是继续扩展播放器控件，而是让听匣适合长期管理和收听大量本地书籍：

1. 原目录内容变化后可以安全更新书籍。
2. 用户可以记录重要时间点。
3. 书籍多起来后仍能快速查找和整理。
4. 睡前听书体验更完整。
5. 不同书籍可以保留各自的播放习惯。
6. 能明确看到章节和整本书的完成状态。

### 成功标准

- 更新目录不会产生重复书籍或无故丢失当前进度。
- 书签可以在重启后恢复并准确跳转。
- 100 本书的书架仍可快速搜索和排序。
- 睡眠定时支持“本章结束”及渐弱暂停。
- 切换书籍时自动恢复该书上次使用的倍速。
- 章节完成状态与当前播放位置保持一致，并允许用户手动修正。

## 3. 核心设计原则

### 3.1 源目录仍是事实来源

- 音频文件继续使用 SAF URI，不复制到应用目录。
- 重新扫描只更新 Room 索引，不修改或删除用户源文件。
- 用户从书架移除书籍时，仍不删除源目录内容。

### 3.2 播放进度与实际收听状态分离

以下概念不能混为一谈：

- 当前播放位置：下次从哪里继续。
- 章节完成状态：用户是否认为该章已经听完。
- 实际收听时长：未来统计功能使用，本阶段暂不实现。

用户手动跳到后续章节时，当前位置可以前进，但不应自动把所有跳过章节标记为已完整听完。

### 3.3 更新目录时优先保留用户数据

文件匹配必须优先保留：

- 当前章节与位置。
- 章节完成状态。
- 书签。
- 自定义章节标题。
- 每本书播放设置。

无法可靠匹配时不得静默映射，应展示变更摘要并给出提示。

### 3.4 Service 继续作为播放状态权威

- `PlaybackService` 是播放进度的唯一持久化写入方。
- UI 不直接持有 ExoPlayer。
- 睡眠定时由 Service 执行，避免 Activity 退后台后失效。

## 4. 数据模型设计

建议数据库从当前 v3 升级到 v4。

### 4.1 Book 新增字段

| 字段 | 类型 | 默认值 | 说明 |
|---|---|---:|---|
| `sortTitle` | String? | null | 可选排序标题，默认使用 title |
| `isPinned` | Boolean | false | 是否置顶 |
| `playbackSpeed` | Float? | null | 本书独立倍速；null 使用全局设置 |
| `autoPlayNext` | Boolean | true | 是否自动连播下一章 |
| `lastScannedAt` | Long | 0 | 最近成功扫描时间 |

第一版如果不做置顶，`isPinned` 可以延后，但建议本次迁移预留每本书播放设置。

### 4.2 Chapter 新增字段

| 字段 | 类型 | 默认值 | 说明 |
|---|---|---:|---|
| `relativePath` | String | fileName | 相对书籍根目录路径 |
| `fileSize` | Long | 0 | 文件大小，用于重新扫描匹配 |
| `stableKey` | String | 计算生成 | 稳定章节标识 |
| `customTitle` | String? | null | 用户自定义标题 |
| `completionState` | Int | 0 | 0 未听、1 收听中、2 已完成 |
| `completedAt` | Long? | null | 完成时间 |

展示标题规则：

```text
customTitle ?: metadataTitle ?: stripExtension(fileName)
```

### 4.3 BookmarkEntity

```kotlin
@Entity(
    tableName = "bookmarks",
    foreignKeys = [
        ForeignKey(
            entity = BookEntity::class,
            parentColumns = ["id"],
            childColumns = ["bookId"],
            onDelete = ForeignKey.CASCADE,
        ),
        ForeignKey(
            entity = ChapterEntity::class,
            parentColumns = ["id"],
            childColumns = ["chapterId"],
            onDelete = ForeignKey.CASCADE,
        ),
    ],
    indices = [Index("bookId"), Index("chapterId")],
)
data class BookmarkEntity(
    @PrimaryKey(autoGenerate = true) val id: Long = 0,
    val bookId: Long,
    val chapterId: Long,
    val positionMs: Long,
    val note: String? = null,
    val createdAt: Long = System.currentTimeMillis(),
)
```

同一章节允许多个书签。可选增加唯一规则防止用户在同一秒内重复点击：

```text
bookId + chapterId + position bucket
```

### 4.4 章节稳定标识

推荐生成规则：

```text
normalize(relativePath) + fileSize + duration bucket
```

匹配优先级：

1. 原始 document URI 完全一致。
2. `stableKey` 一致。
3. 相对路径一致。
4. 文件名、大小、时长三者中至少两项一致。
5. 无可靠匹配，视为新章节。

不能只使用 `fileName`，因为不同子目录可能存在同名文件。

## 5. 增量重新扫描

### 5.1 用户入口

书籍详情页右上角菜单新增：

- 重新扫描目录。
- 重新授权目录。
- 编辑书籍信息（后续小版本）。
- 从书架移除。

### 5.2 扫描流程

```text
读取当前 Book/Chapter
→ 校验 SAF 权限
→ 扫描目录生成 ScannedChapter
→ 计算新旧章节匹配关系
→ 生成变更预览
→ 用户确认
→ Room 事务更新
→ 重建章节偏移索引
→ 若当前正在播放该书，安全刷新播放列表
```

### 5.3 变更类型

```kotlin
sealed interface ChapterChange {
    data class Unchanged(...) : ChapterChange
    data class Renamed(...) : ChapterChange
    data class Added(...) : ChapterChange
    data class Removed(...) : ChapterChange
    data class Ambiguous(...) : ChapterChange
}
```

### 5.4 确认页

扫描完成后展示：

- 新增 N 章。
- 删除 N 章。
- 重命名/移动 N 章。
- 无法确认匹配 N 章。

默认行为：

- 新章节加入对应自然排序位置。
- 已移除章节从数据库删除。
- 已匹配章节保留原 ID，避免书签外键失效。
- 当前章节被删除时，选择最接近的下一章；没有下一章则选择上一章。
- 无歧义变更可一键确认。

### 5.5 正在播放时重新扫描

不得直接替换播放器正在使用的列表。

建议策略：

- 当前章节仍存在：播放不中断，下一次切章前刷新列表。
- 当前章节被删除：允许当前已打开文件继续播放，结束后切到新的相邻章节。
- 用户主动点击新章节：立即加载更新后的播放列表。

## 6. 书签与备注

### 6.1 播放器交互

全屏播放器新增书签按钮：

- 单击：立即保存当前时间点，并显示 Snackbar。
- 保存后 Snackbar 提供“添加备注”。
- 长按或二次入口：直接打开备注编辑框。

### 6.2 书签列表

书籍详情页增加“书签”区块或 Tab：

```text
第 12 章 · 18:35
这里开始解释核心概念
2026-07-16 22:10
```

操作：

- 点击：开始播放对应章节并跳转到时间点。
- 编辑备注。
- 删除。

### 6.3 播放跳转

`PlayerController.playBook()` 增加明确的 `positionMs` 调用路径；只有书签章节与当前书一致时才能直接 seek，否则应重建该书播放列表后定位。

### 6.4 边界条件

- 章节重新扫描匹配成功时书签不变。
- 章节确实被删除时，其书签随外键级联删除；确认页必须提示将删除多少书签。
- position 超过新文件时长时，截断到新时长。

## 7. 搜索、排序与筛选

### 7.1 搜索

书架 TopAppBar 增加搜索入口，搜索范围：

- 书名。
- 作者。
- 可选：章节标题。

查询应防抖约 200–300ms，并由 Room 执行，不在 Compose 中对大量列表反复过滤。

### 7.2 排序

支持：

1. 最近播放。
2. 最近导入。
3. 书名。
4. 收听进度。

置顶书籍始终优先，置顶内部再按当前排序规则排序。

### 7.3 筛选

- 全部。
- 未开始。
- 收听中。
- 已完成。
- 需要重新授权。

书籍状态计算：

```text
未开始：lastPlayedAt == 0
已完成：所有有效章节 completionState == COMPLETED
收听中：其余已开始书籍
```

### 7.4 偏好持久化

DataStore 保存：

- 当前排序方式。
- 当前筛选方式。
- 网格或列表布局。

搜索词不持久化。

## 8. 睡眠定时增强

### 8.1 模式

```kotlin
sealed interface SleepTimerMode {
    data object Off
    data class AfterDuration(val durationMs: Long)
    data object EndOfChapter
}
```

本阶段支持：

- 15 / 30 / 45 / 60 分钟。
- 本章结束。
- 关闭。

### 8.2 本章结束

开启后：

- 当前章节自然播放结束时暂停。
- 不自动进入下一章。
- 用户手动切章后，定时目标更新为新章节结束。
- 用户关闭定时后恢复正常连播策略。

### 8.3 音量渐弱

倒计时模式到期前 30 秒：

- 记录原始播放器音量。
- 每秒平滑降低音量。
- 到期暂停。
- 暂停后恢复原始音量，避免下次播放仍然静音。

如果用户在渐弱期间关闭定时，应立即恢复音量。

### 8.4 延长定时

剩余 60 秒时可通过播放器界面延长 15 分钟。通知栏交互可延后，不作为 0.2 首发阻断项。

### 8.5 状态归属

- 定时逻辑完全位于 PlaybackService。
- UI 通过自定义 SessionCommand 设置模式。
- Service 向 Controller 暴露剩余时间和模式。
- 不要求进程死亡后恢复睡眠定时。

## 9. 每本书播放设置

### 9.1 设置项

0.2 首发支持：

- 独立倍速。
- 是否自动连播下一章。

后续可增加：

- 快退秒数。
- 快进秒数。
- 跳过静音。

### 9.2 优先级规则

```text
Book.playbackSpeed ?: globalDefaultSpeed
```

用户在播放器修改倍速时，默认只更新当前书的倍速。设置页提供“全局默认倍速”，仅影响尚未设置独立倍速的书。

可在倍速菜单提供：

- 设为本书倍速。
- 恢复全局默认。

### 9.3 切书行为

播放新书时，在 `prepare()` 前应用该书速度。章节切换不得重置速度。

## 10. 章节完成状态

### 10.1 状态定义

```kotlin
enum class ChapterCompletionState {
    NOT_STARTED,
    IN_PROGRESS,
    COMPLETED,
}
```

Room 中使用 Int 存储，并通过转换函数映射。

### 10.2 自动状态规则

- 首次播放且 position > 0：`IN_PROGRESS`。
- 自然播放到章节结尾：`COMPLETED`。
- 播放位置达到章节时长的 95%：可以标记完成，但应避免未知时长误判。
- 用户手动跳到下一章：原章节保持 `IN_PROGRESS`，不自动完成。
- 从已完成章节重新播放：默认保持 `COMPLETED`。

### 10.3 手动操作

章节长按菜单：

- 标记为已听。
- 标记为未听。
- 从本章开始播放。

书籍详情菜单：

- 全部标记已听。
- 清除全部完成状态。

批量操作必须二次确认。

### 10.4 书籍完成状态

仅当所有有效章节都为 `COMPLETED` 时，书籍才算完成。

新增章节后，原本已完成的书籍会重新变为收听中，这是预期行为。

### 12.1 可独立测试的策略类

以下逻辑必须从 Android API 中分离：

### 12.2 事务边界

重新扫描确认后的数据库更新必须在单个 Room 事务内：

```text
更新 Book
→ 更新已匹配 Chapter
→ 插入新增 Chapter
→ 删除移除 Chapter
→ 修正当前章节和累计进度
→ 提交
```

SAF 权限获取与释放不能放在数据库事务内部；外部操作失败时需有补偿逻辑。

### 14.3 Media3/真机验证

- 后台播放时本章结束暂停。

## 17. 验收清单

### 增量扫描

- [ ] 增加文件后重新扫描可新增章节。
- [ ] 删除文件后明确提示并安全移除章节。
- [ ] 文件改名后尽量保留章节 ID、进度和书签。
- [ ] 当前章节删除后可以选择合理替代章节。
- [ ] 无变化扫描不会重建整本书。

### 书签

- [ ] 播放器可以一键添加书签。
- [ ] 支持备注编辑与删除。
- [ ] 点击书签可准确跳转。
- [ ] 重启后书签仍存在。

### 书架管理

- [ ] 可搜索书名和作者。
- [ ] 可按最近、导入时间、书名、进度排序。
- [ ] 可筛选未开始、收听中、已完成、需授权。
- [ ] 排序和布局设置重启后保留。

### 播放体验

- [ ] 本章结束睡眠模式不会进入下一章。
- [ ] 倒计时结束前渐弱，暂停后恢复音量。
- [ ] 每本书独立恢复倍速。
- [ ] 手动切章不会错误标记上一章完成。
- [ ] 自然播放结束会标记章节完成。

### 稳定性

- [ ] v1/v2/v3 数据库均可升级到 v4。
- [ ] 更新目录不丢失现有书架和进度。
- [ ] 后台播放、通知栏、蓝牙控制无回归。
- [ ] CI 全部通过并产出可覆盖安装 APK。

## 18. 推荐最终范围

如果需要控制开发周期，0.2 首发应锁定以下五项：

```text
增量重新扫描
+ 书签
+ 搜索/排序
+ 本章结束睡眠
+ 每本书独立倍速
```

章节完成状态可以与增量扫描的数据模型一起落地，但复杂批量操作可以放到 0.2.1。

建议避免在 0.2 同时加入 M4B、Android Auto 或完整统计，否则会同时修改导入模型、播放模型和页面结构，显著增加回归风险。
