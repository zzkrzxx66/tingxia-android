## 1. 产品定位

**听匣**是一款 **本地有声书播放器**，不是音乐 App，也不是在线听书平台。

### 核心承诺

1. 从本地文件夹导入一整本书  
2. 后台持续播放（锁屏 / 通知栏可控）  
3. 自动记住听到哪（断点续听）  
4. 倍速播放 + 睡眠定时  

### 产品边界

| 做 | 不做（第一版） |
|----|----------------|
| 本地文件 / 文件夹导入 | 账号系统 |
| 书架与章节管理 | 云同步 |
| 后台播放与系统媒体控件 | 在线书城 / 版权内容 |
| 进度记忆、倍速、睡眠定时 | 社交、社区 |
| 完全离线、隐私优先 | AI 摘要（后期可选） |

### 与音乐播放器的差异

听书场景强调：**按书组织、章节连播、长会话续听、倍速、睡眠、车载/锁屏操作**。  
第一版交互与数据模型按「书 → 章节」设计，而不是「歌曲列表」。

## 4. 技术栈（已锁定）

| 层级 | 技术 | 说明 |
|------|------|------|
| 语言 | Kotlin | Android 主流 |
| UI | Jetpack Compose | 现代声明式 UI |
| 架构 | MVVM + 清晰模块分层 | UI 薄，data/player 厚 |
| 异步 | Coroutines + Flow | 播放状态、进度流 |
| 依赖注入 | Hilt | 官方推荐 |
| 播放引擎 | **Media3（ExoPlayer）** | 长音频 / 有声书首选 |
| 后台与锁屏 | Media3 Session + Foreground Service | 系统媒体中心、蓝牙、线控 |
| 本地数据库 | Room | 书架、章节、进度 |
| 偏好设置 | DataStore | 倍速默认值、主题等 |
| 文件选择 | SAF（Storage Access Framework） | `OpenDocumentTree` 等 |
| 元数据 | MediaMetadataRetriever（可后续增强） | 标题、时长、内嵌封面 |
| 导航 | Navigation Compose | 书架 → 详情 → 播放器 |

### 主要依赖方向（版本在建工程时锁定最新稳定版）

- `androidx.media3:media3-exoplayer`  
- `androidx.media3:media3-session`

- `androidx.room` + KSP  
- `com.google.dagger:hilt-android`

## 7. 本地导入方案

### 7.1 导入方式

| 方式 | API / 能力 | 优先级 |
|------|------------|--------|
| 选文件夹 | `OpenDocumentTree` | **主推**（一目录 = 一本书） |
| 选多个文件 | `OpenMultipleDocuments` | 补充 |
| 系统打开方式 | `VIEW` / 分享进入 | 体验加分 |

### 7.2 权限与持久化

```text
用户选择目录
  → takePersistableUriPermission()
  → 扫描音频文件
  → 文件名自然排序（numeric-aware）
  → 生成 Book + Chapters
  → 读取时长 / 封面
  → 写入 Room，展示在书架
```

权限丢失时：书籍标记为「需重新授权」，引导用户重新关联目录，而不是静默消失。

### 7.3 文件策略

- **引用源文件 URI，不复制大文件到 App 私有目录**（省空间、导入快）  
- 依赖 SAF 可持久化权限  
- 卸载重装或用户撤销权限后，需重新授权  

### 7.4 支持格式（MVP）

`mp3` / `m4a` / `m4b` / `aac` / `flac` / `ogg` / `wav`  

优先保证常见有声书格式（尤其 **mp3 / m4a / m4b**）播放稳定。

### 7.5 自动成书规则

| 规则 | 行为 |
|------|------|
| 书名 | 默认使用文件夹名 |
| 章节 | 同目录音频文件，按自然序排列 |
| 封面 | 优先同目录 `cover.jpg` / `cover.png` → 其次内嵌 album art → 否则首字母纯色封面 |
| 排序 | 自然排序，避免 `第10章` 排在 `第2章` 前 |

- 启动时恢复「最近在听」

## 9. 播放器架构

```text
UI (Compose)
  → PlayerViewModel
  → PlaybackService (MediaSessionService)
  → Media3 ExoPlayer
  → 章节播放列表（MediaItem 序列）
```

### 必须具备的系统能力

1. **前台服务** + 媒体通知（播放 / 暂停 / 下一章等）  
2. **MediaSession**（系统媒体中心、蓝牙、线控）  
3. **音频焦点**（来电、导航语音抢占时的合理行为）  
4. **BecomingNoisy**（拔耳机自动暂停）  
5. **章节串联播放**（当前章结束自动下一章）  
6. **倍速**（`PlaybackParameters`）  
7. **Seek**（±15s / ±30s，进度条拖动）  

> 禁止仅在 Activity 内持有播放器：锁屏或退到后台易被系统停掉。必须走 Service + MediaSession 标准方案。


## 11. 体验细则（已达成共识的加分项）

1. 打开 App，若有最近在听，突出「继续播放」  
2. 通知栏展示：书名 + 章节 + 封面（能力范围内）  
3. 进度条可拖，拖动时显示时间  
4. 切换章节 **不重置倍速**  
5. 睡眠定时到点建议 **音量渐弱约 30 秒再停**（P1 可完善）  
6. 大目录导入显示扫描进度，主线程不卡顿  
7. 源目录权限失效时明确提示重新授权  
8. 深色模式优先（夜间听书场景）  
9. 控件偏大，方便通勤 / 床上盲操  

### 已知风险与对策

| 风险 | 对策 |
|------|------|
| 后台被系统杀死 | 前台服务 + 正确媒体通知 |
| SAF 权限丢失 | 持久权限 + 失效引导 |
| 文件名乱序 | 自然排序算法 |
| 国产 ROM 激进杀后台 | 设置页说明电池优化 / 自启动（如实提示） |
| 仅 Activity 播放 | 强制 Service 架构 |

## 13. 构建与 GitHub CI

### 结论

当前技术选型 **可以在 GitHub Actions 上构建**，无需改栈。

| 构建位置 | 可行性 | 用途 |
|----------|--------|------|
| 本机 Android Studio | 可以（需 JDK + Android SDK） | 日常开发、真机调试 |
| **GitHub Actions** | **可以** | 自动编译、产出 debug APK、PR 检查 |
| 无 SDK 的服务器环境 | 本地无法直接 assemble | 仍可编写源码并推送由 CI 构建 |

### CI 建议（后续落地）

- 工作流文件：`.github/workflows/android.yml`  
- Runner：`ubuntu-latest`  
- JDK：17（或 21，与工程对齐）  
- 命令：`./gradlew assembleDebug`  
- 产物：上传 `app/build/outputs/apk/debug/*.apk` 为 Artifact  

### Debug vs Release

| 类型 | 签名 | 适用 |
|------|------|------|
| Debug | 默认 debug 密钥 | CI、内测、PR |
| Release | 自有 keystore，放入 GitHub Secrets，**不进仓库** | 正式分发 / 上架 |

### 注意

- 仓库需包含完整 Gradle Wrapper（`gradlew` 等），保证 CI 可复现构建  
- **有声书音频文件不要提交 Git**，只提交源码与文档  
- CI 验证「能编译」；真实「选文件夹听书」仍依赖真机验证
